# CF-M365-Admin

一个面向社区用户与中小团队的 **Microsoft 365（Office 365）账号自助开通与轻量管理面板**，基于 **Cloudflare Workers + Microsoft Graph API** 实现。  
无需服务器、开箱即用，适合用于：测试租户管理、账号发放、内部自助开通、演示环境等。

> 友情提醒：该项目是技术工具，不是“账号分发服务”。部署与使用者需自行确保合规（详见文末免责声明）。

## 🚀 相对于初版的全新重构与增强

相比于仅提供基础“URL token验证”、“单租户硬编码配置”和“粗糙接口”的早期版本，当前 V3 版本进行了脱胎换骨的全面重构和 UI 深度打磨。
以下是 **新增与核心增强** 的重点说明：

### 🔒 架构与安全性大幅升级
- **彻底抛弃硬编码配置**：新增**可视化首次安装向导**。早期版本必须在 Cloudflare 的配置页面或代码中手写海量的 `SKU_MAP` 和环境密钥。现在，所有的全局账户、API密钥等敏感信息，全部通过直观的后台 UI 录入，并加密存入 Cloudflare KV 持久化。
- **放弃明文 URL Token，改用高规格 Session 鉴权**：早期版本只需在网址后面加个 `?token=xxx` 就能强行访问后台，存在极大的泄漏和嗅探风险。新版改用带安全属性的 HttpOnly Cookie 存放会话密钥，并且支持主动点击“退出”以销毁凭证，实现了完整的现代化登录验证闭环。
- **防探测与兜底保护**：新增对未授权、错误请求或扫描脚本的拦截，全部自动重定向回首页或抛出友好的 404 页面，彻底隐藏管理后台的真实地址。
- **防刷集成与风控约束**：前端原生接入 Cloudflare Turnstile 人机验证，结合高危敏感词字典拦截机制（保护 `admin`, `root` 等账号被抢注或误删），从根本上避免黑产脚本的滥用。

### 🏢 多租户（多全局账户）的完美支持
- **多全局可视化管理**：由原本仅支持固定单租户，升级为**无限制多全局挂载**。在后台可以动态增删改查多个全局管理凭证（Tenant ID, Client Secret），系统会自动获取它们对应的订阅配额。
- **全自动的 SKU 智能拉取**：再也不用自己去 Graph Explorer 里辛苦翻找枯燥的 SKU ID 对应关系了！在配置新的全局账户时，只需点击“获取 SKU”，系统即可秒速自动抓取订阅，并在界面显示自动匹配出来的中文名。

### 🛡️ 订阅隔离与授权控制（核心独创功能）
- **精确到具体应用的“阉割”分发**：老版本只能分配整个 SKU 授权。现在的 V3 版，**首创了“细颗粒度权限控制”功能**。无论是在后台直建账号，还是在前台生成邀请码时，你都可以**勾选并剔除掉易被黑产滥用的云盘（OneDrive/SharePoint）、邮局（Exchange）和 Teams 等核心组件**，极大地保护了主租户的存活寿命！
- **带隔离范围的智能邀请码**：重构了邀请码逻辑。现在的邀请码可以精准限制“只能注册 A 全局下的 B 订阅，且禁止使用 OneDrive”。支持批量生成附带固定前缀的纯大写、防混淆格式邀请码，并可视化展示邀请码的剩余额度。

### 🧑‍💻 用户管理台体验翻倍
- **高性能前端与数据管理**：在用户管理界面新增了多重组合检索条件与分页处理（支持根据账号、用户名、全局归属和拥有的 SKU 类别来进行精确的模糊搜索与多级排序）。
- **丝滑的 UI 与排版交互体验**：全面重构了下拉列表框组件，增加了统一的防抖动收起机制与精致的展开/关闭过渡动画；精细化调整了表格列宽占比，对长文本（如禁用的应用名称）做出了恰到好处的截断与折行处理，在移动端和 PC 端都能获得极佳的数据浏览体验。
- **密码重置逻辑强化**：支持批量选中用户执行操作。在重置密码时，管理员现在可以选择“手动录入”和“自动生成高强度合规密码”，更加贴合运维工作流。前端在密码表单也全面落实了“大/小/数/符四选三且长度不低于八位”的实时校验。

## 🧩 V3 升级与迁移指南（必读）

如果你是从早期的单文件（环境变量配置版）升级，请注意以下几点核心架构变更：

### 1) 配置持久化：由 Env Vars 变更为 Cloudflare KV
旧版中通常在 `wrangler.toml` 或面板配置大段 `SKU_MAP` 和 `ADMIN_TOKEN` 环境变量。  
**新版本引入了 `CONFIG_KV`**，一切设置（包括全局账号、后台路径、防刷密钥）全在界面操作并加密存储至 KV。
> 升级建议：保留基本的 `CONFIG_KV` 绑定即可，你可以通过管理后台重新配置你的全局和应用映射，过程将非常顺滑且支持自动获取 SKU。

### 2) 访问控制：鉴权升级与防刷接入
旧版直接使用带 Token 的明文 URL（如 `/?token=xxx`）来访问后台。  
**新版通过安全的后台入口路径（如 `/admin`）登录**，使用用户名/密码及 HttpOnly 的 Session Cookie 进行校验。  
> 同时强烈建议：在后台设置中填入 Cloudflare Turnstile 的 Site Key 与 Secret Key 开启注册防刷。

### 3) 邀请码与权限隔离（重大升级）
早期的订阅开通往往是赋予该订阅下的所有应用权限（包含了 OneDrive, Exchange 等）。  
**新版本增加了“禁用应用权限”功能**。在生成邀请码和后台直接建号时，均可以勾选并剔除掉易被黑产滥用的云盘和邮局组件，保护主租户安全。邀请码页面还全面集成了高级搜索框，支持按照“邀请码”、“限制次数”、“已用”、“状态”以及“限制范围”进行全局模糊搜索过滤，并默认以 10 个数量、1 次限制生成邀请码，极大提高了管理效率。

### 4) 安全防护：404 与智能跳转兜底
所有的前端输入现在都有严格的正则验证和报错反馈；对于所有异常或探测请求，会自动重定向回登录页或抛出干净的 404 Not Found 界面。在管理面板也新增了登出按钮，点击后立即注销 Cookie 防止浏览器残留。

## 🛠️ 部署前提

你需要准备：  
1. **Cloudflare 账号**：用于部署 Workers 和绑定 KV。  
2. **Microsoft 365 全局管理员权限**：用于注册 Azure AD (Entra ID) 应用。  
3. **Azure AD (Entra ID) 应用信息**：
   - 获取 `Client ID`、`Tenant ID`
   - 创建 `Client Secret`（保存其 Value 的值）  
   - 配置 Graph API 权限：添加 **Application permissions** 中的 `User.ReadWrite.All`、`Directory.Read.All`（可选）、`Organization.Read.All`（可选），并执行 **管理员同意 (Grant admin consent)**。

## ⚙️ Cloudflare Workers 部署教程

### 1）创建 KV 命名空间
进入 Cloudflare Dashboard -> Workers & Pages -> KV -> Create namespace  
命名为：`CONFIG_KV`

### 2）创建 Worker 脚本
新建一个 Worker，将仓库中最新的 `worker_v3.js`（或其他主文件）的内容完全复制并粘贴进去。

### 3）绑定 KV
在当前 Worker 的 Settings -> Variables & Secrets -> KV Namespace Bindings 处：
- Variable name 填：`CONFIG_KV`
- KV namespace 选：你刚才创建的那个 KV

### 4）保存并部署
访问该 Worker 分配的域名（例如 `xxx.workers.dev`），将自动进入**首次安装引导页面**。

## 🚀 快速开始（首次体验）

1. **初始化后台**：打开你的域名，在向导中设置管理账号、密码，以及你想要的后台路径（如 `/admin-panel`）。
2. **登录后台**：使用你刚才设置的账号进入。
3. **添加全局账户**：进入「全局账户」，填入你在 Azure 申请的 TenantId、ClientId 和 ClientSecret 等信息。
4. **同步 SKU**：填好信息后，点击弹窗里的“点我获取 SKU”，系统将自动连接微软拉取你的订阅列表。点击保存。
5. **开启防护（可选）**：前往「设置」配置 Turnstile 密钥，开启人机校验。开启邀请码模式。
6. **分发与使用**：前往「邀请码管理」，为指定订阅生成专属邀请码分发给用户。用户在首页输入后即可极速注册！

## 🧯 排错建议（Troubleshooting）

- **后台 404 / 进不去**：大概率是你忘记了你自定义的后台路径，或者 Session 过期了。重新通过你在安装时填写的路径（如 `/admin`）进入。
- **“点我获取 SKU”获取失败**：先确认 TenantId / ClientId / ClientSecret 三项是否有多余的空格，以及在 Entra ID 中是否给予了正确的 API 权限并**同意了管理员授权**。
- **拉取不到可禁用的应用服务**：只有在生成邀请码或后台建号时，单选某**一个**订阅才能正确匹配和拉取它旗下包含的 Service Plans。
